## Context

- ID 009 (long press): `use-long-press.ts` já limpa seleção (`getSelection().removeAllRanges`)
  e expõe `data-long-press-target`. `globals.css` bloqueia `user-select` nesses alvos e
  preserva inputs/editor. `/notas/page.tsx` usa `useLongPress` em 5 pontos (ContentRow/Card,
  pastas). `bulk-actions.tsx` (`ItemContextMenuContent`) já mostra header com tipo + título +
  tags do item (não só mobile). → majoritariamente pronto, validar.
- ID 025 (menu de notas): `notas/page.tsx` já consolidou em kebab `aria-label="Ações da pasta"`
  + `FolderMenu` (favoritar, ocultar/limpar concluídos, reordenar, renomear, mover, AGENTS.md,
  copiar link, excluir). Topo só com "Novo item" + kebab. → pronto, validar.
- ID 051 (busca): `api/items/search/route.ts` já busca em `contentMd`, e o endpoint retorna o
  item completo (inclui `contentMd`). `topbar.tsx > SearchResults` mostra só título + data. Falta
  exibir trecho/snippet com destaque do termo. Sem mudança de API: snippet client-side.
- ID 052 (expansão): `heading-collapse-extension.ts` guarda posições recolhidas só em memória
  (perde ao remontar). Persistir por nota sem schema → localStorage por `itemId`, restaurando
  após o conteúdo carregar (TipTap `setContent` é assíncrono).
- ID 022 (calendário fullscreen): `calendar-grid.tsx` no modo `googleLike` limita a 2 eventos no
  desktop (`entryIndex >= 2 ? lg:hidden`) mesmo com células altas (`fillHeight`, 6 linhas 1fr),
  causando má distribuição/"mais N" precoce. Mobile usa o mesmo grid dentro de `lg:hidden`, então
  o ajuste desktop não afeta mobile.
- ID 054 (modais): `quick-capture.tsx` (criar/editar tarefa e nota), `calendar-event-capture.tsx`
  e o overlay de tarefa em `item-detail.tsx` JÁ seguem `modais-standalone.html` (radius 28, glass
  `bg-white/.90` + blur 24, header com ícone + eyebrow + close 32px, inputs h-52 radius-18, footer
  pílula gradiente). Faltam os utilitários: `dialog.tsx`, `shortcut-help-modal.tsx`,
  `agents-editor-modal.tsx` e o `EventSheet` (editar evento em `calendar-board.tsx`), que usam
  `rounded-xl` branco opaco e botões quadrados.

## Decisions

- Decision: 051 via snippet client-side (sem mudar API). Reason: o endpoint já retorna `contentMd`;
  evita mudança de payload/contrato. ADR needed: no.
- Decision: 052 persiste em localStorage por `itemId` (`doit-note-collapse:<id>`), restaurando após
  o conteúdo carregar. Reason: view-state por dispositivo, sem schema nem writes no servidor a cada
  toggle. ADR needed: no.
- Decision: 022 usa `ResizeObserver` diretamente no componente `CalendarCard` em `page.tsx` que é o fullscreen verdadeiro. Reason: O limite precisa ser calculado em relação à altura exata renderizada na tela, subtraindo os elementos adjacentes. ADR needed: no.
- Decision: 054 alinha só os modais utilitários; não reescreve os já conformes. Reason: evitar
  regressão em fluxos centrais já validados. ADR needed: no.

## Risks

- Risk: persistência de expansão por posição ordinal pode divergir se a estrutura de títulos mudar.
  Mitigation: mapear por índice ordinal do título e ignorar índices fora do range ao restaurar.
- Risk: medir altura de célula no fullscreen pode dar 0 antes do layout. Mitigation: ResizeObserver
  + fallback de limite mínimo.
