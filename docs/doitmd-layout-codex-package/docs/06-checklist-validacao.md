# Checklist de validação

## Antes de implementar

- [ ] Confirmar rotas atuais.
- [ ] Confirmar componentes existentes reutilizáveis.
- [ ] Confirmar modelo de `Item`.
- [ ] Confirmar estrutura de `Pastas`.
- [ ] Confirmar dados de calendário e Drive.

## Durante implementação

- [ ] Não criar `projects` ou `areas`.
- [ ] Não duplicar lógica de Item.
- [ ] Não importar banco em client component.
- [ ] Preservar SWR/hooks existentes.
- [ ] Manter acessibilidade básica.
- [ ] Manter performance no mobile.

## Validação visual

- [ ] Desktop 1440px.
- [ ] Desktop 1280px.
- [ ] Mobile 390px.
- [ ] Mobile 412px.
- [ ] Tema claro.
- [ ] Tema escuro, se suportado.

## Validação funcional

- [ ] Criar novo Item.
- [ ] Capturar no Inbox.
- [ ] Marcar item como concluído.
- [ ] Abrir nota/editor.
- [ ] Filtrar por pasta.
- [ ] Abrir evento do calendário.
- [ ] Ver estado de sync/auditoria.
