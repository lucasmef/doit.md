# Prompt para Codex

Você está trabalhando no projeto `doit.md`.

Há um pacote de referência visual em `doitmd-layout-codex-package/` com HTMLs estáticos para desktop e mobile.

Objetivo: planejar uma implementação incremental para aproximar a interface real do app desses mockups bento/glassmorphism, preservando a arquitetura atual.

Antes de alterar código:

1. Leia `docs/01-contexto-produto.md`.
2. Leia `docs/02-diretrizes-visuais.md`.
3. Leia `docs/03-mapeamento-rotas.md`.
4. Leia `docs/04-plano-implementacao.md`.
5. Leia `docs/05-mobile.md`.
6. Abra os HTMLs de referência.
7. Inspecione o projeto real.
8. Gere um plano técnico incremental.

Restrições:

- Não introduzir `projects` ou `areas`.
- Usar `pastas` como agrupador visual.
- Manter `Item` como entidade central.
- Não copiar os HTMLs diretamente para produção.
- Reusar componentes existentes sempre que possível.
- Primeiro planejar, depois implementar.
- Para alterações visuais, validar com navegador e screenshots.

Entregável inicial esperado:

- Diagnóstico da UI atual.
- Mapeamento de mockup -> rotas/componentes reais.
- Plano incremental em etapas pequenas.
- Riscos técnicos.
- Ordem recomendada de implementação.
